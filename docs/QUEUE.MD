## 1. Conventions

### 1.1 Kind (work type)

Every row uses exactly one **Kind**.

| Kind | Meaning |
|------|---------|
| **fix** | Wrong behavior, bug, contract break, or missing wiring in **existing** code (`IL-*`). |
| **feature** | **New** user- or operator-visible capability. |
| **change** | **Improvement** to existing behavior ÔÇö not a bug, not a whole new surface. |
| **hardening** | Production readiness: security, signing, guardrails. |
| **platform** | Infra, CI, compose, packaging. |
| **reliability** | Ops: runbooks, smokes, distribution notes. |
| **spike** | Time-boxed research. |
| **tech_debt** | Refactors. |
| **compliance** | Release **gates** (`HG-*`). |
| **deferred** | Out of scope for now (`FB-N*`). |

Legacy **Type** column in ┬ž2 tables maps to Kind: FeatureÔćĺ**feature**, HardeningÔćĺ**hardening**, PlatformÔćĺ**platform**, ReliabilityÔćĺ**reliability**, SpikeÔćĺ**spike**.

### 1.2 ID prefixes

| Prefix | Scope |
|--------|--------|
| **FB-** | Roadmap, epics, deferred |
| **IL-** | Fixes only (**fix** kind) |
| **HG-** | Compliance gates |

### 1.3 Priority, phase, category, status

**Priority:** HIGH / MEDIUM / LOW (maps to P0ÔÇôP3).

**Phase (for `FB-*` open work):** **A** ÔÇö core trading intelligence; **B** ÔÇö operator product; **C** ÔÇö distributed platform; **D** ÔÇö deferred.

**Category:** **FR**, **PF**, **UI**, **DS**, **MS**, **XX** (roadmap); **EX**, **MD**, **DP**, **RT**, **CP**, **OB**, **OP**, **CI**, **CFG** (fixes / ┬ž3).

**Fix urgency (`IL-*`):** **P1** / **P2** / **P3**.

**Status:** Open / In progress / Done.
