# Decision engine

> **Architecture status:** This file describes the **repository as implemented today**. For **target** system architecture during the canonical APEX migration, see [`../CANONICAL_SPEC_INDEX.MD`](../CANONICAL_SPEC_INDEX.MD) and [`../Human Provided Specs/new_specs/canonical/`](../Human%20Provided%20Specs/new_specs/canonical/). Where this document conflicts with canonical APEX specs on **future** design, treat this file as **as-built** only, not the roadmap.


## Shared tick

- **`run_step.py`** ‚Äî **`run_decision_tick`**: single step used by **`app/runtime/live_service`** and **`backtesting/replay.py`**. Records decision latency metric when wired.

## Pipeline

- **`app/contracts/decision_snapshots.py`** ‚Äî **FB-CAN-015:** Pydantic models for **Market / Structural / Safety / Execution feedback / Service config** input families (APEX data-contract spec); validated before each tick.
- **`app/contracts/snapshot_builders.py`** ‚Äî **FB-CAN-015:** builds **`DecisionBoundaryInput`** from the legacy feature row + `mid_price`, `spread_bps`, `data_timestamp`, `AppSettings`; merges canonical freshness/reliability scalars into the effective feature dict; full bundle JSON is stored under **`ForecastPacket.forecast_diagnostics["canonical_boundary_input"]`**.
- **`state_engine.py`** ‚Äî **FB-CAN-004:** builds APEX **`CanonicalStateOutput`** from `ForecastPacket` + features (5-class regime probs, confidence, transition, novelty, heat, reflexivity, degradation); attached to **`RegimeOutput.apex`** and merged into **`RiskState`** for sizing.
- **`trigger_engine.py`** ‚Äî **FB-CAN-005:** Setup ‚Üí Pre-Trigger ‚Üí Confirmed Trigger; **`TriggerOutput`** serialized under **`ForecastPacket.forecast_diagnostics["trigger"]`**; gates proposals in **`run_spec_policy_step`** when trigger invalid.
- **`auction_engine.py`** ‚Äî **FB-CAN-006:** scores long/short/flat candidates (canonical weighted score + penalties), selects winner under top-N and notional cap; result in **`forecast_diagnostics["auction"]`**.
- **`pipeline.py`** ‚Äî `DecisionPipeline`: regime + forecast + route + `ActionProposal` from features and risk state
- **`features_live.py`** ‚Äî Live tick / overlay feature row construction
- **`feature_frame.py`** ‚Äî Bar enrichment and merges (live/replay)
- **`spec_policy_proposal.py`** ‚Äî PolicySystem + trigger + auction ‚Üí `ActionProposal` (replaces legacy standalone route‚Üíproposal helper removed in FB-CAN-019)
- **`audit.py`** ‚Äî `decision_trace` structures for logging and optional QuestDB persistence

## Parity rule

Any change that affects trading behavior should keep **`run_decision_tick`** behavior consistent for **live** and **replay** unless explicitly documented otherwise.
